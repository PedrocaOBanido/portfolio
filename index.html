>_ ENGINEERING PHILOSOPHY

Building correct, observable, and pragmatic software.

I prefer simple systems with explicit responsibilities, observable behavior, and abstractions justified by recurring problems — not by pattern books.

Architecture When Needed

The right architecture emerges from understanding the problem, not from applying a template. I design boundaries that serve the domain.

Boundaries Enable Speed

Well-defined contracts and clear ownership let teams move fast safely. Good boundaries are the foundation of maintainable systems.

Clarity Over Ceremony

Abstractions are justified when they reduce cognitive load, not when they add indirection. If a pattern doesn't clarify the system, it doesn't belong.

AI for Acceleration, Human for Ownership

AI speeds up exploration and boilerplate. Humans own planning, testing, review, and architectural decisions. Never commit code you don't understand.

Code-First Exploration

When approaching unfamiliar systems, I read the code and tests, trace a request end-to-end, then ask targeted questions. The code is the source of truth.

Documentation and Pairing

I invest in writing things down and pairing with engineers who have less context. A well-explained system pays back in fewer mistakes and faster autonomy.

Case Study 1

Event-Driven Payment Systems 💳

Payments

Problem: Mobile app polled for Pix payment status — delayed UI updates, wasted bandwidth, scaling pressure on backend at 100M+ user scale.

Solution: Replaced polling with Kafka-based event streaming. Backend emits state-change events; Android receives push notifications and refreshes only when needed.

Key decisions: Kafka for event transport, REST for data fetch, event versioning for out-of-order delivery, idempotency keys for deduplication.

Impact: Fewer network requests, faster state transitions, reduced backend load, fewer stale-state errors.

Tech: Kotlin, Android, Java, Spring Boot, Kafka, PostgreSQL, AWS

Case Study 2

On-Device Semantic Search 🔍

Privacy

Problem: Traditional keyword search couldn't handle natural language queries for device settings, files, and Samsung native app content (gallery, etc.).

Solution: Hybrid retrieval — Lucene for keyword indexing + TFLite embedding model for semantic similarity. All on-device.

Key decisions: Quantized model for latency, pre-computed embeddings at indexing time, two-stage retrieval (fast Lucene → precise semantic), SentencePiece BPE + custom token encoding, zero network calls.

Impact: Improved search relevance over keyword-only baseline, fully privacy-preserving architecture, works offline.

Tech: Kotlin, Android, Lucene, TensorFlow Lite, SentencePiece BPE
